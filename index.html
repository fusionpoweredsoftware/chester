What is Chester?

Chester is a simple but powerful web-based chess interface that bridges the gap between human gameplay and programmatic chess testing. Built around the industry-leading Stockfish AI engine, Chester provides both an intuitive drag-and-drop chess board for visual play and a comprehensive REST API for automated chess operations. Whether you're developing chess software, conducting AI research, preparing for tournaments, or simply want to play against a world-class chess engine with full configurability, Chester delivers a seamless experience that adapts to your workflow.

Quick Start

Getting Chester running takes less than a minute with the automated setup script that handles all dependencies and asset downloads:

chmod +x chester.sh && ./chester.sh

The script automatically starts the server at http://localhost:8001 where you'll find the complete chess interface with real-time engine configuration, move history tracking, and instant synchronization between UI and API interactions.

API Endpoints

Chester exposes a clean REST API for programmatic chess control, perfect for automation, testing, and integration with larger chess systems:

POST /move - Execute chess moves using algebraic notation
POST /computer - Trigger Stockfish AI to calculate and play
POST /undo - Undo the last move played
POST /new - Start a fresh chess game
GET /state - Retrieve current position in FEN notation
GET /history - Get complete move history as JSON

Example API usage for making moves and triggering computer play:

curl -X POST http://localhost:8001/move \ -H "Content-Type: application/json" \ -d '{"from":"e2","to":"e4"}' curl -X POST http://localhost:8001/computer

Command Line Interface

Chester generates executable shell scripts in the commands/ directory for quick chess operations from your terminal:

# Make moves ./commands/move.sh e2 e4 ./commands/move.sh g1 f3 # Let computer play ./commands/computer.sh # Game control ./commands/undo.sh ./commands/new.sh # Check status ./commands/state.sh # Current position ./commands/history.sh # Move list
